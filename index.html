About & Syntax

About

Blank is a Markdown note-taking app that renders structured content as interactive checklists. Notes are organized into collapsible sections, each with a completion checkbox. Includes progress tracking and full-text search.

By Essential Software · v2.0

Markdown syntax

Blank uses a custom Markdown dialect. Use the syntax below to structure your notes.

Document structure

### Phase 1 — Title
### B0 — Setup
### Final Steps
Top-level section block. Any text after ### is valid — the label is displayed as-is.
## Day 1 — Description
## Week 2
## Foundation
Sub-section inside a top-level block. Any text after ## is valid (unless it contains “Troubleshooting” or “Remarks”, which trigger special blocks).
# Any heading here
Checkable step block. Any # heading that doesn’t contain Troubleshooting or Remarks becomes a checkable step. The heading text can be anything.
## Troubleshooting
Yellow collapsible block — triggered when the heading contains “Troubleshooting”
## Remarks
Teal collapsible block with speech bubble icon — triggered when the heading contains “Remarks”. Use for notes, asides, or commentary.
#### Sub-heading inside a step
Plain #### headings render as section headings inside step body content
## Critical Reminders
## Sources
End-of-document block. Critical Reminders renders red with a warning icon. Sources renders blue with a globe icon — suited for reference or link lists.

Tags

[tags]: Supabase, Node.js, CLI
Comma-separated tags shown on the note card. Place anywhere in the document (typically near the top, outside any section heading). Use the exact [tags]: prefix.

Images

[image]: https://example.com/photo.jpg
[image]: https://… "Caption text"
Loads an image directly from a URL. The optional quoted caption appears below the image. Images load lazily and show a fallback message if the URL fails. Only web URLs are supported — no local files.

Category

[category]: Coding
Assigns the note to a homepage category tab. Must exactly match one of the available category names (case-insensitive). If it doesn't match, the property is ignored and the note appears under All. Place near the top, outside any section heading.

Overview / Introduction

Any content before the first ### Phase heading
Rendered as the collapsible Overview block at the top of the document body. Supports all standard markdown: paragraphs, lists, tables, headings, code blocks.

Phase / Chapter naming rules

### Phase 1 — Accounts
### B0 — Foundation
### Final Steps
### Deployment
Any ### line opens a top-level section — the text after ### can be anything. Inside it, ## lines are sub-section blocks and # lines are step/point blocks.

Privacy Policy

Effective 6 April 2026

Blank stores your progress, edits, and preferences in the device's local storage only. This data never leaves the device and is not accessible to Essential Software.

Everything you write or import stays on the device and is never transmitted or stored remotely. Notes can embed images from external URLs — when this happens, the app fetches the image directly from that URL. No personal data is involved, and Essential Software has no control over third-party image hosts.

When you are signed in on an account, the app connects to Essential Software servers to sync notes across devices and verify access. Only your encrypted note data and an account identifier are transmitted — no behavioural or personal data.

Progress can be cleared using the Reset button in the progress bar. All app data can be removed by clearing local storage in your device settings.

Sign in to sync

Your notes, progress, favorites, and edits will sync across all your devices automatically.

Session expired

Your session has expired. Sign in again to continue syncing — your local notes are safe.

Delete your synced data?
Your notes and progress will be permanently removed from our servers. Your local data on this device is kept.
